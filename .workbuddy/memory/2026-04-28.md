## 任务：v0.2 开发 — 流式响应 + Worktree 隔离 + Committee 模式

### 流式响应
- `ChatParams` 新增 `stream?: boolean` + `onTextDelta?: (text: string) => void`
- `BaseAnthropicAdapter.chat()` 在 `stream=true` 时内部用 streaming API（`chatViaStream`），仍返回 `Promise<ChatResponse>`
- 修复 `chatStream()` 缺失 tool_use 事件：新增 content_block_start/delta/stop 处理
- `ChatStreamChunk` 新增 `toolCalls?: ToolCall[]` 字段（message_stop chunk 携带完整工具调用）
- `FallbackExecutor.execute()` 透传 stream 参数
- `AgentLoopDeps` 新增 `onStreamText?: (text: string) => void`
- CLI `Orchestrator.execute()` 传入 `onStreamText: (text) => process.stdout.write(text)`

### Git Worktree 隔离
- 新增 `src/agent/worktree-manager.ts`
  - `acquireWorktree(mainDir, agentType)` → 创建 detached HEAD worktree → 返回 `{ worktreeDir, cleanup }`
  - `resolveIsolation(mainDir, requested?)` → 非 git repo 自动降级为 context
  - worktree 存放在 `.git/orchestrator-worktrees/<agentType>-<ts>-<rand>/`
- `AgentLoop.spawnSubAgent()` 集成：
  - 检查 subDef.isolation → worktree 时创建隔离目录
  - 创建失败时 fallback 到 context
  - 子 Agent 完成后自动 cleanup

### Committee 模式
- 新增 `src/agent/committee.ts`
  - `Committee` 类：多 Agent 并行执行同一任务
  - 三种聚合策略：`concat`（拼接）/ `majority`（多数表决）/ `best`（最长输出）
  - 内部 Semaphore 控制并发数
  - `CommitteeResult` 含 status/content/members/strategy/totalCost/totalSteps
- CLI 新增 `committee` 命令：
  - `multi-agent committee <task> -a explore,coder,reviewer -s concat`
  - 支持 `--agents`, `--strategy`, `--budget` 参数

- ChatParams 新增 `onRetry?: (attempt, error) => void`

- AgentLoop 传入 `onRetry` 回调，在流式输出中插入 `[retry attempt N: error]` 标记
- 用户现在能看到重试标记，不会以为输出是连续的

### 限制 2：单次调用可能超预算
- CostTracker 新增 `estimateWorstCase(provider, inputEstimate, maxTokens)` 方法
- 用 `max_tokens` 做最坏情况输出 token 预估
- AgentLoop 在每次模型调用前做前置检查：`canAfford(worstCaseCost)`
- 超预算时直接返回 `budget_exceeded`，不再发起调用
- AgentDefinition 新增 `maxTokensPerStep` 字段
- AgentLoop 传入 `ChatParams.maxTokens = definition.maxTokensPerStep ?? 4096`
- AgentLoop 新增 `estimateInputTokens()` 辅助方法（4 chars/token 估算）

- 新增 `pruneStaleWorktrees(mainDir)` 函数，启动时自动清理上次残留

### 限制 4：cheerio 可选依赖

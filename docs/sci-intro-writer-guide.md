# SCI 论文绪论写作指南

## 概述

本系统提供了一个专门的 agent (`sci-intro-writer`) 和配套 workflow (`sci-introduction`)，用于撰写 SCI 论文第一章绪论（Introduction）。该系统支持通过谷歌学术（Google Scholar）和知网（CNKI）进行文献检索与综述写作。

## 安全设计

### Agent 权限限制

`sci-intro-writer` agent 遵循**最小权限原则**，其工具权限配置为：

| 工具 | 权限 | 原因 |
|------|------|------|
| Read | allow | 读取项目文件（如阅读已有草稿） |
| Grep | allow | 搜索文件内容 |
| Glob | allow | 查找文件 |
| WebSearch | allow | 在谷歌学术和知网检索文献 |
| WebFetch | allow | 获取论文摘要和详情 |
| MemoryRead/Write/Search | allow | 保存和管理文献知识 |
| **Write** | **deny** | 不具备写入文件权限（安全考虑） |
| **Edit** | **deny** | 不具备编辑文件权限（安全考虑） |
| **Bash** | **deny** | 不具备执行命令权限（安全考虑） |
| task | deny | 作为 specialist agent，不委托任务 |

这种设计确保了：
- Agent 只能生成内容，不能随意修改项目文件
- 防止意外的文件系统操作
- 职责单一，专注于内容生成

## 使用方法

### 方法 1: 直接调用 Agent

```
task → sci-intro-writer: "我正在写一篇关于[研究主题]的SCI论文，目标期刊是[期刊名]。请帮我撰写第一章绪论，包含研究背景、文献综述、研究空白、研究目标和论文结构。用英文撰写，预期字数3000字。"
```

### 方法 2: 使用 Workflow（推荐）

Workflow 提供了结构化的三步流程：

1. **文献检索与分析** — 在谷歌学术和知网检索相关文献
2. **绪论撰写** — 基于文献分析结果撰写完整绪论
3. **质量检查** — 验证逻辑、引用、语言等维度

#### 启动 Workflow

```
workflow run sci-introduction --variable researchTopic="基于Transformer的时序预测方法" --variable language="english" --variable wordCount="3000"
```

#### Workflow 变量说明

| 变量 | 默认值 | 说明 |
|------|--------|------|
| researchTopic | ""（必填） | 研究主题/论文题目 |
| targetJournal | "SCI" | 目标期刊类型 |
| language | "english" | 写作语言：english/chinese |
| wordCount | "3000" | 预期字数 |

## 文献检索策略

### 谷歌学术（Google Scholar）

**关键词组合模式：**
- `[核心概念] review` — 寻找综述文章
- `[核心概念] state of the art` — 了解最新进展
- `[方法] in [领域]` — 特定方法的应用
- `[研究主题] methodology` — 方法学相关

**检索优先级：**
1. 近5年（2020-2025）高被引论文
2. 领域内顶刊（Nature, Science, IEEE, ACM等）
3. Review/Survey 文章（快速了解领域全景）

### 知网（CNKI）

**关键词组合模式：**
- `[中文关键词] 综述` — 中文综述文章
- `[中文关键词] 研究现状` — 国内研究进展
- `[中文关键词] 方法 技术` — 技术方法相关

**检索优先级：**
1. 核心期刊（北大核心、CSSCI、CSCD）
2. 博士学位论文（绪论部分通常有全面的文献综述）
3. 近5年文献

## 质量标准

Workflow 会在最后一步进行以下检查：

| 维度 | 检查内容 |
|------|----------|
| 逻辑链条 | 背景→问题→综述→目标→结构是否层层递进 |
| 引用完整性 | 每个关键论断是否有文献支撑 |
| 文献覆盖度 | 是否覆盖了领域核心文献和前沿文献 |
| 语言质量 | 学术用语是否规范，是否存在口语化表达 |
| 结构比例 | 各部分篇幅是否符合标准比例 |
| 字数符合度 | 总字数是否接近预期 |

## 注意事项

1. **网络访问**：文献检索依赖网络访问，确保能够访问谷歌学术和知网
2. **文献质量**：部分论文可能需要机构订阅，agent 会使用公开可获取的摘要信息
3. **学术诚信**：所有引用必须基于真实检索到的文献，禁止编造
4. **人工审核**：生成的内容需要人工审核和润色，确保符合目标期刊的具体要求

## 与前一轮方案的对比

### 改进点

| 方面 | 前一轮方案 | 本方案 |
|------|------------|--------|
| **工具权限** | Write/Edit: allow（安全隐患） | Write/Edit: deny（遵循最小权限原则） |
| **Workflow 步骤** | 6 步（过于复杂） | 3 步（精简实用） |
| **Workflow 依赖** | 使用不存在的 `dependsOn` 字段 | 使用项目原生的 `${steps.id.content}` 变量替换 |
| **与项目集成** | 独立 YAML 文件 | 完全集成到现有项目结构 |
| **验证机制** | 无 | 通过 parser 自动验证 YAML 格式 |

### 核心优势

1. **安全性**：移除了文件写入权限，防止意外修改
2. **实用性**：3 步流程符合实际使用场景，降低使用成本
3. **兼容性**：完全遵循项目现有的 agent 和 workflow 架构
4. **可维护性**：代码与现有模式一致，易于后续维护
